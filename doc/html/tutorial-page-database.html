Step 1: Database

Tables

Our tutorial application will use a database called sdbc_tutorial. It consists of two tables login and news.

The login table contains information that is necessary for a user to log in. Only logged in users are allowed to add news entries. The table has this layout:

fielddescriptions
idthe user id
namea user name for the login
passwordthe (encrypted) password of the user

The second table news contains all news messages the system knows and will display. It has a structure like this:

fielddescriptions
idid of the entry
titlea short description of the message
submitteda timestamp that documents when the message was added
textthe message text

Setting up the Database

The tutorial folder contains SQL scripts for all supported database systems. If you are trying to run this tutorial on a MySQL database use the file called mysql.sql or postgre.sql if you are running a postgreSQL database.

You only need to load the included scripts into a database called sdbc_tutorial. This is the database that all scripts of the example web application will try to connect to. Make sure you also have a database user called sdbc_tutorial because the scripts also try to connect with this username and the password sdbc1234. If you are rewriting the tutorial - what's by the recommended to understand the examples much better - you don't need to bother with these details.
