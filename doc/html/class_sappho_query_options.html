Detailed Description

This class is used to set any statement options like WHERE clauses or ORDER BY etc.

This class represents any kind of options that aplly to a statement. This may range from WHERE clauses to ORDER BY columns or subselects and joins.

andWhere ( $  field,
$  operation = '',
$  value = '' 
)

Add an AND condition to the WHERE clause.

The use of this method extends the defined WHERE clause with a AND clause. So before calling this method you need to initialize a WHERE clause with where().

You may either give all three parameters to define a simple condition like AND column = 'value' or the like.

If the first parameter is another SapphoQueryOptions object a new subcondition like AND (column = 'value' OR column_b = 1 ...) will be added. So this is a convinient way to bild nested WHERE conditions.

Parameters:
$fieldthe column that has to match
$operationa match operation
$valuethe value the column has to match
Returns:
modified object
getClause ( $  table)

Get all clauses set with this options.

By using this method all conditions and options defined with this object are evaluated and all set clauses are generated. There is a fixed order for the generated clauses:

  1. WHERE
  2. ORDER BY

It is necessary to provide the table that the query will be executed on to do correct datatype escaping by using the tablecache of the assigned connection.

Parameters:
$tablethe table the query will be executed on
Returns:
all conditions and clauses in a string

Construct a valid WHERE clause.

By using this method the stored options are processed and a valid WHERE clause is built and returned. It is already correctly escaped for use with the assigned connection. This method is mainly designed to be used internally but you may call it to get the WHERE clause separatly.

It is necessary to provide the table that the query will be executed on to do correct datatype escaping by using the tablecache of the assigned connection.

Parameters:
$tablethe table the statement will be executed on
$subcontitionset to true if used during getWhereClause() if used in nested conditions
Returns:
the correctly escaped WHERE clause as string
